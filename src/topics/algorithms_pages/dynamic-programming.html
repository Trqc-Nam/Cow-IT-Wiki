Một Vài Bài Toán Quy Hoạch Động Thú Vị

Kỹ thuật ghi nhớ và tái sử dụng kết quả bài toán con — nền tảng của mọi kỳ thi học sinh giỏi Tin học.

📋 Mục lục

  • 📌 1. Khái niệm DP
  • 📌 2. Đặc điểm
  • 📊 3. Bảng phương án
  • 💻 4. Ví dụ minh hoạ

1. Quy hoạch động là gì?

Quy hoạch động (Dynamic Programming – DP) là một kĩ thuật thuật toán dùng để giải quyết bài toán phức tạp bằng cách chia nó thành các bài toán con nhỏ hơn, lưu trữ kết quả để tối ưu quá trình tính toán tổng thể.

Thay vì tính đi tính lại cùng một kết quả nhiều lần, DP "ghi nhớ" (memoize) các kết quả đã tính vào một bảng, giúp giảm độ phức tạp từ hàm mũ xuống còn đa thức.

📸 So sánh trực quan đệ quy ngây thơ và DP cho bài toán Fibonacci.

2. Đặc điểm nhận dạng bài toán DP

  • 🔁 Bài toán con gối nhau (Overlapping Subproblems): Cùng một bài toán con được giải nhiều lần.
  • 🎯 Cấu trúc con tối ưu (Optimal Substructure): Lời giải tối ưu của bài toán lớn được xây dựng từ lời giải tối ưu của bài toán con.

3. Bảng phương án mẫu — Bài toán Túi ba lô (Knapsack)

dp[i][j] = Giá trị tối đa khi xét i đồ vật đầu tiên với sức chứa j:

i \ j 0 1 2 3 4 5
0 (Không đồ vật) 0 0 0 0 0 0
1 (W=2, V=6) 0 0 6 6 6 6
2 (W=2, V=10) 0 0 10 10 16 16
3 (W=3, V=12) 0 0 10 12 16 22

4. Ví dụ: Fibonacci với DP Bottom-up

// Độ phức tạp: O(n) time, O(n) space
int fibonacci(int n) {
    int dp[n+1];
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2]; // Công thức truy hồi
    }
    return dp[n];
}
// fibonacci(10) = 55
💡 Tư duy cốt lõi của DP: "Thay vì hỏi 'Tôi có thể làm gì từ đây?', hãy hỏi 'Tôi đã đến đây bằng cách nào?' và lưu lại câu trả lời."
