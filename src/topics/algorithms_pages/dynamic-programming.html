Một Vài Bài Toán Quy Hoạch Động Thú Vị

1. Quy hoạch động là gì?

Quy hoạch động (Dynamic Programming – DP) là một kĩ thuật thuật toán dùng để giải quyết một bài toán phức tạp bằng cách chia nó thành các bài toán con nhỏ hơn, lưu trữ kết quả của các bài toán con này để tối ưu quá trình tính toán tổng thể.

Thay vì tính đi tính lại cùng một kết quả nhiều lần (như đệ quy ngây thơ), DP "ghi nhớ" (memoize) các kết quả đã tính vào một bảng (mảng), giúp giảm độ phức tạp từ hàm mũ xuống còn đa thức.

2. Đặc điểm nhận dạng bài toán DP

Một bài toán phù hợp với DP thường có 2 tính chất sau:

  • 🔁 Bài toán con gối nhau (Overlapping Subproblems): Cùng một bài toán con được giải nhiều lần. Nếu lưu kết quả lại, ta tránh tính lại.
  • 🎯 Cấu trúc con tối ưu (Optimal Substructure): Lời giải tối ưu của bài toán lớn được xây dựng từ lời giải tối ưu của các bài toán con.

3. Bảng phương án mẫu

Bảng dưới đây minh hoạ bảng phương án cơ bản của quy hoạch động dạng mảng 2 chiều dp[i][j]. Trong bài toán Túi ba lô (Knapsack), dp[i][j] là giá trị tối đa khi xét i đồ vật đầu tiên với sức chứa j:

i \ j (Sức chứa) 0 1 2 3 4 5
0 (Không đồ vật) 0 0 0 0 0 0
1 (W=2, V=6) 0 0 6 6 6 6
2 (W=2, V=10) 0 0 10 10 16 16
3 (W=3, V=12) 0 0 10 12 16 22

📊 Bảng DP bài toán Túi ba lô 0/1 với 3 đồ vật và sức chứa tối đa 5.

4. Ví dụ: Bài toán Fibonacci với DP

Bài toán Fibonacci là ví dụ kinh điển nhất để minh họa sự khác biệt giữa đệ quy thông thường và DP: 

// DP Bottom-up: Tính Fibonacci
// Độ phức tạp: O(n) time, O(n) space
int fibonacci(int n) {
    int dp[n+1];
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2];  // Công thức truy hồi
    }
    return dp[n];
}
// fibonacci(10) = 55
// Không tính lại bất kỳ giá trị nào!
💡 Tư duy cốt lõi của DP: "Thay vì hỏi 'Tôi có thể làm gì từ đây?', hãy hỏi 'Tôi đã đến đây bằng cách nào?' và lưu lại câu trả lời."
