Từ bài toán 7 cây cầu Königsberg của Euler đến thuật toán Dijkstra — nền tảng của Google Maps và mọi hệ thống định tuyến.

1. Lý thuyết đồ thị là gì?

Lý thuyết đồ thị là một nhánh của toán học rời rạc, chuyên nghiên cứu về các mạng lưới kết nối. Nó ra đời vào thế kỷ 18 từ một câu đố thú vị: "Làm sao để đi qua đúng 7 cây cầu của thành phố Königsberg, mỗi cầu đúng 1 lần và trở về điểm xuất phát?"

Nhà toán học Leonhard Euler đã giải quyết bài toán này bằng cách trừu tượng hoá: Gạt bỏ mọi chi tiết thừa, chỉ giữ lại Đỉnh (ngã tư) và Cạnh (con đường). Ngay lúc đó, Lý thuyết đồ thị ra đời!

📸 Bài toán 7 cây cầu Königsberg — khởi nguồn của Lý thuyết đồ thị.

"Bản chất của vũ trụ không nằm ở bản thân các vật thể, mà nằm ở cách chúng kết nối với nhau."

Một Đồ thị G = (V, E):

  1. Đỉnh (Vertex – V): Các thực thể độc lập (ngã tư, máy tính, tài khoản Facebook...).
  2. Cạnh (Edge – E): Mối liên kết giữa 2 Đỉnh. Vô hướng (bạn bè Facebook) hoặc có hướng (follow Instagram).

2. Phân tích: Tìm đường đi ngắn nhất (Dijkstra)

Bạn đang ở ngã tư S và muốn đến ngã tư D trong thời gian ngắn nhất. Dijkstra dùng thao tác Lan truyền (Relaxation):

// Nếu đi từ S→u→v nhanh hơn đường cũ từ S→v:
if (dist[u] + w < dist[v]):
    dist[v] = dist[u] + w

3. Bảng chạy tay (Trace Table)

Đồ thị 5 đỉnh (0–4), xuất phát từ đỉnh 0. Cạnh: 0→1:4, 0→2:1, 2→1:2, 1→3:1, 2→3:5, 3→4:3

Bước dist[0] dist[1] dist[2] dist[3] dist[4] Thao tác
Khởi tạo 0 ∞ ∞ ∞ ∞ dist[0]=0
Duyệt đỉnh 0 0 4 1 ∞ ∞ 0→1:4; 0→2:1
Duyệt đỉnh 2 0 3 1 6 ∞ 2→1: 1+2=3✓; 2→3:6
Duyệt đỉnh 1 0 3 1 4 ∞ 1→3: 3+1=4✓
Duyệt đỉnh 3 0 3 1 4 7 3→4: 4+3=7
✅ Kết thúc 0 3 1 4 7 Lộ trình: 0→2→1→3→4 = 7

4. Code C++ — Dijkstra với Priority Queue

#include <vector>
#include <queue>
using namespace std;
const int INF = 1e9;

void dijkstra(int start, int n, vector<vector<pair<int,int>>>& g) {
    vector<int> dist(n, INF);
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;
    dist[start] = 0;
    pq.push({0, start});

    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;  // Đã có đường tốt hơn
        for (auto [v, w] : g[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}

5. Tổng kết

Lý thuyết đồ thị là một cách nhìn nhận thế giới. Việc trừu tượng hoá vấn đề thực tế thành Đỉnh và Cạnh chính là bước chuyển mình từ Coder thành Kỹ sư Hệ thống thực thụ.
